# Design Tokens

Our design system is built on a comprehensive set of design tokens that ensure consistency across all components. These tokens are defined in CSS custom properties and automatically adapt to light and dark themes.

## üé® Color System

### Primary Colors
Our primary color palette is based on a blue color scheme that works well in both light and dark themes.

```css
/* Primary Colors */
--ds-colors-primary-50: #f0f9ff;
--ds-colors-primary-100: #e0f2fe;
--ds-colors-primary-200: #bae6fd;
--ds-colors-primary-300: #7dd3fc;
--ds-colors-primary-400: #38bdf8;
--ds-colors-primary-500: #0ea5e9;
--ds-colors-primary-600: #0284c7;
--ds-colors-primary-700: #0369a1;
--ds-colors-primary-800: #075985;
--ds-colors-primary-900: #0c4a6e;
--ds-colors-primary-950: #082f49;
```

### Semantic Colors
Semantic colors provide meaning and context to UI elements:

```css
/* Background Colors */
--ds-semantic-background-primary: var(--ds-colors-neutral-50);
--ds-semantic-background-secondary: var(--ds-colors-neutral-100);
--ds-semantic-background-tertiary: var(--ds-colors-neutral-200);

/* Text Colors */
--ds-semantic-text-primary: var(--ds-colors-neutral-900);
--ds-semantic-text-secondary: var(--ds-colors-neutral-700);
--ds-semantic-text-tertiary: var(--ds-colors-neutral-500);

/* Border Colors */
--ds-semantic-border-primary: var(--ds-colors-neutral-300);
--ds-semantic-border-secondary: var(--ds-colors-neutral-200);
--ds-semantic-border-focus: var(--ds-colors-primary-500);
```

### Status Colors
Colors that communicate different states and meanings:

- **Success**: Green colors for positive actions and states
- **Warning**: Yellow/amber colors for caution
- **Danger**: Red colors for errors and destructive actions
- **Info**: Blue colors for informational content

The design system automatically adapts to dark mode using CSS custom properties:

```css

  :root {
    --ds-semantic-background-primary: var(--ds-colors-neutral-900);
    --ds-semantic-background-secondary: var(--ds-colors-neutral-800);
```

## üîß Customization

### Adding New Tokens
To add new design tokens:

1. Add the token to `src/theme/theme.json`
2. Update the CSS custom properties in `src/theme/tokens.scss`
3. Document the token in this file
4. Add examples to Storybook

### Theme Overrides
To override theme values:

```css
:root {
  --ds-semantic-text-primary: #your-color;
  --ds-spacing-md: 20px;
}
```
